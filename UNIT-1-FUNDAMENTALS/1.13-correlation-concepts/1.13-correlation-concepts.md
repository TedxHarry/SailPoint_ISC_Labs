## 📚 CORE CONCEPTS

### What is Correlation?

**Definition:** Correlation is ISC's process of matching accounts across different systems to the same person (identity).

**Problem it solves:** Multiple systems have accounts for the same person but use different identifiers.

**Example:** Morgan Chen
- Entra ID: morgan.chen@contoso.com
- HRIS: Employee ID E-00547
- Finance App: Username "mchen"
- QuickBooks: Username "morgan.chen.finance"

These are four different account names. Correlation realizes they're all Morgan Chen.

---

### How Correlation Works

**Automatic Correlation**

ISC applies rules to match accounts. Common rules:

1. **Email match:** Entra ID email = HRIS email → same person
 - Morgan Chen: Entra ID email = morgan.chen@contoso.com, HRIS email = morgan.chen@contoso.com → MATCH

2. **Name + Department match:** First name + last name + department → same person
 - Morgan Chen Finance (HRIS) + Morgan Chen (Entra ID with department=Finance) → MATCH

3. **Employee ID match:** HRIS employee ID appears in other systems
 - HRIS: E-00547 for Morgan Chen, Entra ID notes E-00547 → MATCH

4. **Username + domain:** Username matches across systems
 - mchen@finance.app + mchen@hris → MATCH (if configured)

**Precedence:** Rules applied in order. First match wins. If email matches, that's the correlation. If email fails, try name+dept, etc.

---

### Orphaned Accounts and Identities

**Orphaned Account:** Account exists in a system but ISC can't correlate it to an identity.

**Cause:** Account created before ISC, or account creator left system, or correlation rule doesn't match.

**Example:** Finance app has account "jsmith" that doesn't match anyone in Entra ID or HRIS.
- ISC knows: jsmith exists in Finance app
- ISC doesn't know: Who is jsmith?
- Result: Orphaned account (can't manage it, can't revoke it)

**Orphaned Identity:** Identity in ISC (created from HRIS, Entra ID) that has no accounts in target systems.

**Cause:** New hire not yet provisioned, or all accounts deleted but identity still exists.

**Example:** Casey Kim is hired (created in HRIS and Entra ID) but ISC hasn't provisioned Finance app account yet.
- ISC identity: Casey Kim created
- ISC accounts: No Finance app account yet
- Result: Orphaned identity until provisioning completes

---

### Correlation Configuration

**Configure for each system:**

1. **Priority:** Which system is authoritative for identities? (Usually HRIS)
 - HRIS identity is primary → other systems match to it

2. **Matching attributes:** What attributes define a match?
 - Email: morgan.chen@contoso.com
 - Name + Department: First + Last + Dept
 - Employee ID: E-00547

3. **Correlation rule syntax:** "Match on: email. If email fails, match on: name + department"

**Example (Contoso):**
```
Correlation Rules:
1. If email matches → same identity
2. Else if (first_name + last_name + department) matches → same identity
3. Else if employee_id matches → same identity
4. Else → orphaned account
```

---

### Correlation Problems

| Problem | Impact | Solution |
|---------|--------|----------|
| **Email mismatch** | Account can't be correlated (email changed, typo) | Manual correlation or fix source data |
| **Duplicate accounts** | Same person has multiple accounts in same system | Merge/dedup in source system or ISC |
| **Wrong correlation** | Account matched to wrong person | Manual recorrelation or rule adjustment |
| **Orphaned accounts** | Can't manage account, can't revoke access | Manual correlation or accept as unmanaged |
| **Name changes** | Account created under old name, person renamed | Manual correlation |

**Example:** Morgan Chen marries, name changes to Morgan Smith.
- Entra ID updated: morgan.smith@contoso.com
- Finance app not updated: Still mchen
- Correlation breaks: Email doesn't match
- Result: Finance app account becomes orphaned
- Solution: Manual correlation (tell ISC mchen is Morgan Smith) or update Finance app

---

### Manual Correlation

When automatic fails, ISC allows manual correlation. Administrator maps account to identity:

"jsmith account in Finance app = Casey Kim"

Then ISC treats them as correlated and manages the account.

---

## 🧠 KEY TAKEAWAYS

- **Correlation = matching accounts across systems to the same identity**
- **Automatic correlation uses rules (email, name+dept, employee ID)**
- **Orphaned accounts:** Can't be correlated, can't be managed
- **Orphaned identities:** Person exists in ISC but no accounts provisioned yet
- **Manual correlation:** When automatic fails, administrators manually map accounts to identities

## 🎓 CERTIFICATION ALIGNMENT

**Question 1:** Morgan Chen has an Entra ID account (morgan.chen@contoso.com) and an HRIS record. Morgan's Finance app account is "mchen". ISC's correlation rule is "Match on email". What happens?

A) ISC correlates Finance app account to Morgan Chen
B) ISC creates duplicate identity for Finance app account
C) ✅ Finance app account remains orphaned (email "mchen" doesn't match)
D) ISC asks user to confirm

**Answer: C.** Email rule doesn't match (mchen ≠ morgan.chen@contoso.com). Correlation fails. Solution: Add name-based rule or manual correlation.

---

**Question 2:** After account discovery, ISC shows 150 accounts in Finance app but only 140 identities in ISC. What does this indicate?

A) ISC failed to import some accounts
B) ✅ 10 accounts are orphaned (can't be correlated to any identity)
C) ISC has duplicate identities
D) Accounts don't need to be correlated

**Answer: B.** More accounts than identities means some accounts don't match any identity. These are orphaned.
