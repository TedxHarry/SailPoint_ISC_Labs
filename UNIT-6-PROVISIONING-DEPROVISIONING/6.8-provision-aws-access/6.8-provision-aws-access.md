### TASK 7: Troubleshooting

**Issue 1: MFA Setup Failed**

```
Error: User12 can't set up MFA

Cause:
в”ңв”Җ Authenticator app not installed
в”ңв”Җ QR code not scanned properly
в”ңв”Җ Time sync issue between phone and AWS
в””в”Җ Virtual MFA device already assigned

Solution:
1. Download authenticator app:
 в”ңв”Җ Google Authenticator (iOS/Android)
 в”ңв”Җ Authy (iOS/Android)
 в”ңв”Җ Microsoft Authenticator (iOS/Android)
 в””в”Җ Any TOTP app

2. Time sync:
 в”ңв”Җ Check phone time is correct
 в”ңв”Җ Enable automatic time sync on phone
 в””в”Җ Try again

3. Reset MFA:
 в”ңв”Җ AWS admin: Go to User12 > Security
 в”ңв”Җ Delete old MFA device
 в”ңв”Җ User12: Retry setup
 в””в”Җ Should succeed

4. If still failing:
 в”ңв”Җ Contact AWS support
 в”ңв”Җ Or: Admin manually set MFA
```

**Issue 2: IAM User Can't Assume Role**

```
Error: User12 tries to switch to another role but denied

Cause:
в”ңв”Җ Role trust policy doesn't allow user12-devops
в”ңв”Җ User not in correct group to assume role
в””в”Җ Session duration too short

Solution:
1. Check role trust policy:
 в”ңв”Җ AWS > IAM > Roles > [RoleName]
 в”ңв”Җ Trust policy should include: user12-devops principal
 в”ңв”Җ If missing: Add user to trust policy
 в””в”Җ Save

2. Retry: User12 can now assume role вң“
```

**Issue 3: Provisioning Timeout**

```
Error: "AWS API call timed out"

Cause:
в”ңв”Җ AWS API slow
в”ңв”Җ Rate limit hit
в””в”Җ Network connectivity issue

Solution:
1. Wait , retry
2. Increase provisioning timeout
3. Contact AWS support if API errors continue
```
