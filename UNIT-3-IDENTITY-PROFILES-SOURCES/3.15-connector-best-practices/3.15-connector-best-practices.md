Module 3.14: Other Connector Types.

---

## 📚 CORE CONCEPTS

### Security

**Never commit credentials:** Client secrets, passwords should NOT be in git

**Use secure storage:** Store credentials in vaults (password managers, AWS Secrets, Azure Key Vault)

**Rotate secrets:** Regenerate client secrets periodically

**Minimal permissions:** Connector should only have permissions it needs

---

### Naming Conventions

**Clear names:** "Contoso_Entra_ID" (clear what it is)

**Avoid:** "Source1", "Connector_Test", vague names

---

### Documentation

Document for each connector:
- What system it connects to
- What data it reads/writes
- Authentication method
- Polling frequency
- Dependencies (identity profile, mappings)
- Troubleshooting contacts

---

### Monitoring

- Check connector status regularly
- Monitor aggregation success rates
- Alert on failures
- Review logs for errors

---

### Testing

- Test in dev/test environment first
- Verify all attributes flow correctly
- Check data quality
- Test error scenarios (cred failure, network issues)

---

## 🧠 KEY TAKEAWAYS

- **Security:** Store credentials securely, never hardcode
- **Naming:** Use clear, descriptive names
- **Documentation:** Document everything for troubleshooting
- **Monitoring:** Regular checks prevent issues
