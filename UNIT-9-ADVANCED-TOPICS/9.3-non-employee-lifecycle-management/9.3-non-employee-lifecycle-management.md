## Key Concepts

### Non-Employee Types

| Type | Duration | Access Level | Risk |
|------|----------|--------------|------|
| **Employees** | Ongoing | Full | Managed via HR |
| **Contractors** | 3-12 months | Task-specific | Medium |
| **Vendors** | Service-specific | Limited to service | Medium |
| **Partners** | Ongoing | Limited collaboration | Low-Medium |
| **Contingent** | Temporary | Basic access | Low |
| **Consultants** | Project-based | Task-specific | Medium-High |

### Why Non-Employee Management Matters

**Security Risks:**
- Forgotten access after contractor leaves
- Overprivileged access for short-term need
- No clear owner responsible for access
- Difficult to track across contractor types

**Business Challenges:**
- Contractors often have tight deadlines
- Onboarding processes slow down projects
- Access removal is often forgotten
- Compliance teams struggle with non-standard identities

**ISC Solution:** Treat non-employees as first-class citizens with specific governance

### Non-Employee Access Control Principles

1. **Time-Bounded:** All non-employee access has explicit expiration
2. **Task-Specific:** Access limited to required systems only
3. **Approved:** Manager approval required, with business justification
4. **Restricted:** SoD rules apply to prevent fraud
5. **Monitored:** All non-employee activity audited closely
6. **Automated:** ISC auto-deprovisioning on departure

## Lab: Non-Employee Lifecycle Management

### Lab Objectives

- Create contractor identity in ISC
- Set time-bound access
- Apply SoD restrictions
- Implement access reviews
- Perform deprovisioning

### Lab Steps

#### Step 1: Create Non-Employee Identity

**Action:** ISC Administration → Identities → New Non-Employee

**Configuration:**
```
First Name: Taylor
Last Name: Smith
Type: Contractor
Employment Start: 2026-04-01
Employment End: 2026-06-30 (90 days)
Department: Finance
Title: Senior Analyst
Manager: Morgan Chen
Contractor Company: Tech Solutions Inc.
Agreement Date: 2026-03-20
```

**Expected Result:** Identity created with "Non-Employee" flag

#### Step 2: Request Access

**Action:** Access Requests → New Request

**Configuration:**
```
For: taylor_smith (Contractor)
Request Type: Non-Employee Initial Onboarding
Systems Needed:
  ☑ Finance App (analyst role)
  ☑ ISC (read-only)
  ☑ VPN Access
  ☐ HR Systems
  ☐ Executive Reports

Duration: 2026-04-01 to 2026-06-30 (90 days)
Business Justification: "Needed for Q2 financial analysis project"
Manager: Morgan Chen
```

**Expected Result:** Request submitted for approval

#### Step 3: Approval Workflow

**Action (as Morgan Chen):** Approvals Dashboard

**Review:**
```
Requestor: Morgan Chen
Contractor: Taylor Smith
Duration: 90 days (reasonable for project)
Systems: Finance + ISC + VPN (appropriate)

SoD Check:
  ✓ Analyst role = no approvals
  ✓ Read-only ISC = no modifications
  ✓ VPN = no sensitive data access
  ✓ No conflicts detected

Action: APPROVE
Comment: "On schedule for Q2 project"
```

**Expected Result:** Access approved, provisioning begins

#### Step 4: Apply SoD Restrictions

**Action:** Non-Employee SoD Rules → Taylor Smith

**Rules Applied:**
```
Rule 1: Cannot approve financial transactions
Rule 2: Cannot access payroll data
Rule 3: Cannot export reports outside organization
Rule 4: Cannot access customer PII
```

**Expected Result:** All SoD rules applied automatically

#### Step 5: Configure Auto-Renewal Reminder

**Action:** Non-Employee Management → Auto-Renewal Settings

**Configuration:**
```
For: Contractor type
Send reminder: 30 days before expiration
Notify: Manager (Morgan Chen)
Options: Extend / Deactivate
Manager must respond in: 14 days
Auto-action if no response: Send escalation
```

**Expected Result:** Renewal settings saved

#### Step 6: Simulate Access Review (Day 30)

**Action:** ISC Governance → Non-Employee Review

**Review Notification (May 1):**
```
"Non-Employee Access Review Due"
Contractor: Taylor Smith
Current Access: Finance App, ISC, VPN
Expiration: 2026-06-30 (60 days remaining)

Manager Confirmation:
  [1] Does Taylor still need this access?
  [2] Expected departure date?
  [3] Any concerns?

Manager Response:
  [1] Yes, project ongoing
  [2] June 30 (on schedule)
  [3] None - performing well

ISC Action: Continue access, no changes needed
```

**Expected Result:** Review completed, access continues

#### Step 7: Modify Expiration Date

**Action:** Non-Employee Management → Edit Taylor Smith

**Scenario:**
```
Current expiration: 2026-06-30
New information: Project extending by 10 days
New expiration needed: 2026-07-10

Change Request:
  Current: 2026-06-30
  New: 2026-07-10
  Justification: "Project delayed, wrapping up by July 10"

Manager Approval: Morgan Chen approves
Finance Approval: Finance Manager approves (PO extension)
```

**Expected Result:** Expiration updated to 2026-07-10

#### Step 8: Simulate Deprovisioning

**Action:** Simulate Date → 2026-07-10

**Automatic Deprovisioning Triggered:**

ISC system time reaches July 10, 5:00 PM:
```
Deprovisioning Workflow Executes:
  ✓ Finance App access revoked
  ✓ ISC login disabled
  ✓ VPN certificate revoked
  ✓ Building badge access disabled
  ✓ Contractor marked as inactive

Notifications Sent:
  ✓ Manager: "Taylor Smith deprovisioned"
  ✓ IT: "Remove from all systems"
  ✓ Security: "Disable badge access"

Timeline Recorded:
  [17:00] - Automation triggered
  [17:02] - All access removed
  [17:05] - Notifications sent
  [17:08] - Audit completed
```

**Expected Result:** Automatic deprovisioning completed

#### Step 9: Review Audit Trail

**Action:** ISC Audits → Non-Employee Activity → Taylor Smith

**Audit Timeline:**
```
2026-04-01 17:30 - Identity created (contractor)
2026-04-01 18:00 - Access provisioned
2026-04-02 09:00 - First login recorded
2026-04-15 14:23 - Accessed Finance app (analyst query)
...
2026-07-08 16:45 - Last login recorded
2026-07-10 17:02 - All access revoked (expiration)
2026-07-10 17:03 - Audit completed
```

**Expected Result:** Full audit trail available for compliance
